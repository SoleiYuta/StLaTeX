# UI Design Skill

Use this skill when designing or reviewing StLaTeX user-facing screens.

## Product Feel

StLaTeX should feel:

- strategic
- calm
- analytical
- trustworthy
- focused

Avoid noisy visuals, playful gamification, generic chatbot layouts, and cluttered dashboards.

## Dashboard Priority

The Dashboard is the product. It should immediately show:

- This week's main goal.
- Priority subjects.
- Priority units.
- Temporarily deprioritized units.
- Risk alerts.
- AI reasoning summary.

## Information Hierarchy

- Put the current action at the top.
- Group rationale below the action.
- Show risk as measured guidance, not alarm.
- Make discard units visible but carefully worded.
- Avoid long paragraphs in cards.

## Copy Rules

- Say "今週は優先度を下げる" instead of "捨てる" in student-facing UI unless the context needs the stronger strategic framing.
- Avoid promising admission probability accuracy.
- Use direct, specific reasons such as frequency, current accuracy, remaining days, or score impact.

## Visual Direction

- Dark-mode friendly.
- Clear cards with restrained color.
- Strong typography hierarchy.
- Simple charts only when they clarify a decision.
- Use compact layouts that support repeated daily use.

## MVP Screens

- Dashboard must be polished enough for demo.
- Strategy screen can be text-heavy but must be scannable.
- Study log and mock exam input should be fast, not comprehensive.
